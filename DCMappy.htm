DCMappy - Documentation

This is the documentation for the DCMappy class, designed for Dreamcast.
It is based on original documentation of CDXMappy.

DCMappy handles .FMP files created by Mappy Win32 Editor, (C) Robin Burrows. The current official Mappy Homepage is: http://www.geocities.com/SiliconValley/Vista/7336/robmpy.htm, with mirrors at http://www.rbsite.freeserve.co.uk/robmpy.htm and (if all else fails) http://surf.to/mappy

DCMappy is based on original sources from Robin Burrows, was ported and modified for CDX by Ioannis Karagiorgos and then ported and modified for SDL by Regis Quercioli and then ported and modified for the Dreamcast (PVR) by BB Hood with help from BlackAura, BlueCrab, Quzar, and OneThirty8. 

Known problems:

  • 15 bbp maps:

Are NOT supported. Only 8 bit, 16 bit, 24 bit, and 32 bit maps are supported.

Restrictions:

-          Tiles width and height have to be a multiple of 2. (ex 8x8, 16x16, 32x32, 16x32, 8x16, etc). Same goes for parallax image.

-          Parallax images can only be RGB565 16 bit *.kmg images (No point for an alpha channel).

-          A maximum of four different 8 bpp maps can be loaded at one time.  This is because the Dreamcast only has enough palette entries for four different 8 bbp maps (using the format: PVR_PAL_ARGB8888).  

-          DCMappy only supports FMP ver 0.5 and nothing else.

-          No hexagonal or isometric map support… sorry

 

Generating a Parallax image:

      Place a png image with the correct dimension restrictions in the kmgenc folder. Have the image named “parallax.png”.  Double click on “kmgenc.bat”.

Compiling:

      Use the included makefile as a template for your programs that use DCMappy. Just add onto it.

 

Tutorial

Here is a little code tutorial on how to use DCMappy in your projects: 

Initializing

First of all you have initialize the Dreamcast PVR with certain parameters.  Copy & Paste this code above your main function:

            pvr_init_params_t params = {

                /* Enable opaque, punch-through, and translucent polygons with size 16 */

                { PVR_BINSIZE_16, PVR_BINSIZE_0, PVR_BINSIZE_16, PVR_BINSIZE_0, PVR_BINSIZE_16 },        

                /* Vertex buffer size 512K */

                512*1024

            };

Next, Copy & Paste this code in the beginning of your main function:

            /* Init PVR API */

            if (pvr_init(&params) < 0)

                    return -1;

 

Now, create your DCMappy object: 

    DCMappy * Map = new DCMappy();

Next, load your map: 

    if( Map->LoadMap("Map.fmp" , 320 , 200, BLACK )  == -1 )
        printf("Could not load map.fmp!" ); 

When the map loading fails, it returns -1. LoadMap wants to have the filename of the map, the visible map dimensions (width then height) and the color you want to be transparent(Color Keying).  You have a choice of any color you desire to use as a colorkey but there are 2 predefined colors (BLACK, MAGENTA).  To use your own color you will need to us the function “makecolor16(red, green, blue)”.  For example, if you want white to be transparent you will load the map like this:

    if( Map->LoadMap("Map.fmp" , 320 , 200, makecolor16(255, 255, 255) )  == -1 )
        printf("Could not load map.fmp!" ); 

If you want a parallax surface, call 

   if( Map->CreateParallax( "parallax.kmg" ) == false )
        printf( "Could not load parallax bitmap parallax.bmp" ); 

Thats all there has to be done for initialization. 

Drawing

Here are the functions needed to draw the map.  Remember that DCMappy uses the PVR to draw so you will have to pay attention to where you

use certain draw functions.  Remember that things need to be drawn in this order: Opaque(PVR_LIST_OP_POLY)->Punch Through(PVR_LIST_PT_POLY)->

Transparent(PVR_LIST_TR_POLY).

 

            Map->MapUpdateAnims(); and Map->MapMoveTo(x,y);  should both be called before drawing anything.

 

            Map->DrawParallax() draws the parallax layer.  Can only be called when PVR_LIST_OP_POLY polygons are being submitted. (Between

            pvr_list_begin(PVR_LIST_OP_POLY); and the following pvr_list_finish();). Should not be called along with MapDrawBG.

 

            Map->MapDrawBG() draws the background layer.  Can only be called when PVR_LIST_OP_POLY polygons are being submitted. (Between

            pvr_list_begin(PVR_LIST_OP_POLY); and the following pvr_list_finish();)


            Map->MapDrawBGT() draws the background layer with transparency (using the color key).  Can only be called when

            PVR_LIST_PT_POLY polygons are being submitted. (Between pvr_list_begin(PVR_LIST_PT_POLY); and the following pvr_list_finish();)

 

            Map->DrawFG(N) draws the foreground layer with transparency.  N should be a value between 0 and 2. Start with 0 and work your way to 2 if you

            have 3 foreground layers (ex. Map->DrawFG(0); Map->DrawFG(1); Map->DrawFG(2);). Can only be called when PVR_LIST_PT_POLY

            polygons are being submitted. (Between pvr_list_begin(PVR_LIST_PT_POLY); and the following pvr_list_finish();)

 

// Typical Game Loop

while ( 1 ) {

          pvr_wait_ready();

          pvr_scene_begin();

          pvr_list_begin(PVR_LIST_OP_POLY);

         Map->MapUpdateAnims();

         Map->MapMoveTo(x,y); 

         Map->DrawParallax();

         pvr_list_finish();

         pvr_list_begin(PVR_LIST_PT_POLY);  

         Map->MapDrawBGT();

         Map->MapDrawFG(0);

         Map->MapDrawFG(1);

         pvr_list_finish();

                     pvr_list_begin(PVR_LIST_TR_POLY);

         // Draw Characters, other sprites etc (stuff that is drawn on TOP of the map)

                     pvr_list_finish();

         pvr_scene_finish();

           }
 

Map Methods you should care about

int LoadMap (char * Filename , int Width , int Height, uint16 Key);

Loads the map with the path and name specified into memory so it can be used. This function must be used before calling other Mappy functions. 

Returns 0 on success, -1 on failure. 

Width/Height are the visible size of the map on the screen. Key is the color you want to use for color keying.  There are already 2 predefined colors you can use:

BLACK or MAGENTA.  You can also use a color of your own using the function: “makecolor16(red, green, blue)”.  Check DCMappy.h to see how to use that

function. 

int MapGenerateYLookup ( void );

            Calling this may speed up searching for tiles for functions like MapGetBlock(x,y).  This function should be called after initializing a map.

 

            Returns 0 on success, -1 on failure.  


            void MapUpdateAnims ( void );

            Increments animations. Generally used once during an update loop before drawing. 

 

int MapDrawBG ( void );

            MapDrawBG draws the background layer without transparency, so it will completely overwrite the specified area (ie MapDrawBG() and DrawParallax() both

            should not be called during the same game loop because MapDrawBG() will just draw over the parallax). Can only be called when PVR_LIST_OP_POLY

            polygons are being submitted.

int MapDrawBGT ( void );

            MapDrawBGT draws the background layer with transparency. Can only be called when PVR_LIST_PT_POLY polygons are being submitted.

. 

 

int MapDrawFG (int layer);

            MapDrawFG draws the foreground layer with transparency. layer should only be values 0 through 2 so you have a total of 3 foreground layers.  Can only be

 

            called when PVR_LIST_TR_POLY polygons are being submitted.

int MapChangeLayer (int layernumber);

This changes to a different map layer, returns -1 if failed, or the new layer number (0 to 7) if successful. 

int MapGetTile (int x, int y);

Returns the content of a block at position x,y. The x and y paramaters are the offset from the left and top of the map in BLOCKS, NOT pixels. 

If return value is positive, the cell contains a block. If return value is negative, the cell contains animation. 

BLKSTR * MapGetBlock (int x, int y);

Returns a BLKSTR pointer, useful for collision detection and examining a block structure. Note: the x and y paramaters are the offset from the left and top of the map in BLOCKS, NOT pixels. Example: 

BLKSTR * myblkstr;
myblkstr = MapGetBlock (xoffinblocks, yoffinblocks);
if (myblkstr->tl) { top left collision is set for this block } 

BLKSTR * MapGetBlockLayer (int x, int y, int layer);

           Same as function above, except the user can specify the LAYER he/she wishes to examine.

void MapSetBlock (int x, int y, int strvalue);

The x and y parameters are the offset from the left and top of the map in BLOCKS, NOT pixels. If strvalue is positive, the cell is set to that block structure. If strvalue is negative the cell is set to that anim structure - 1 (i.e. if you want to put anim 3 in, strvalue would be -4). 

 

void MapSetLocation(int x, int y);

   

           The x and y parameters are the offset from the left and top of the screen in pixels.  This sets to were you want to start drawing.  For example, say you call Map-

           >MapSetLocation(320,240); before a draw function.  If you draw the map will show on the bottom right of the screen.  By default it’s location is 0,0 (Top left of the

          screen).

short int GetMapBPP( void );

This call returns the color depth of the current map. 

short int GetMapWidth( void );

This call returns the width of the map in pixels. 

short int GetMapHeight( void );

This call returns the height of the map in pixels. 

short int GetMapWidthInBlocks( void );

This call returns the width of the map in blocks. 

short int GetMapHeightInBlocks( void );

This call returns the height of the map in blocks. 

short int GetMapBlockWidth( void );

This call returns the width of one map block, in pixels. 

short int GetMapBlockHeight( void );

This call returns the height of one map block, in pixels. 

void MapMoveTo( int x , int y );

This call moves the map output to offset x,y. It takes into account invalid coordinates, and will modify them accordingly so as to not scroll off the edges of the map and cause a crash. 

int MapGetXPosition( void );

This call returns the current X offset of the map, in pixels, from the upper-left hand corner. 

int MapGetYPosition( void );

This call returns the current Y offset of the map, in pixels, from the upper-left hand corner. 

int DrawParallax( void );

This call draws the parallax surface on the selected surface.

bool CreateParallax( void );

This call creates and load a parallax surface from *.kmg file Filename
Return value = true if it was successful, otherwise false.
